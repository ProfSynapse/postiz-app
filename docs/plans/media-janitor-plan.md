## Summary

A cron-driven NestJS service in `apps/cron/` that reclaims Railway volume storage by performing a two-phase delete (soft → 7-day grace → hard) against `Media` rows whose published-post window has aged past 7 days. Operates only when `STORAGE_PROVIDER === 'local'`, defaults to dry-run + disabled, and surfaces every action through structured JSON logs + Sentry. Path-traversal and TOCTOU resurrection are handled by layered defenses ratified by all seven consulted specialists.

**Operational posture**: safe-by-default (ENABLED=false, DRY_RUN=true), reversible during grace, observable on every action, contained to the local-storage backend.

## User-Locked Policy (authoritative — driver of every downstream decision)

| Decision | Locked value | Implementation impact |
|---|---|---|
| Storage backend | Railway local volume only | Janitor short-circuits when `STORAGE_PROVIDER !== 'local'` |
| "Posted" definition | At least one `Post` with `state='PUBLISHED'` | Eligibility predicate condition #5 |
| Age clock anchor | Time since **POSTED**, not uploaded | Driven by `MAX(Post.publishDate) FILTER (state='PUBLISHED')` |
| Multi-post clock | **Most recent** PUBLISHED reference | `MAX(publishDate)` across the published-refs set |
| Grace period | 7 days (both ageDays AND graceDays) | `MEDIA_JANITOR_AGE_DAYS=7`, `MEDIA_JANITOR_GRACE_DAYS=7` |
| `User.pictureId` / `SocialMediaAgency.logoId` refs | **NEVER delete** | Predicates #2 and #3; hard-delete re-checks them in-txn |
| Non-PUBLISHED post refs (DRAFT / QUEUE / ERROR) | **BLOCK deletion** | Predicate #4 (`nonPublishedCount = 0`) |
| Historical-thumbnail breakage after hard-delete | Accepted | Backend-coder MUST add null-guard in `posts.service.ts::updateMedia` (R5 bug) |
| Race during grace (re-reference) | **Path A: RESURRECT** (`deletedAt=NULL` + WARN log) | Hard-delete re-checks predicates inside txn; on failure → `UPDATE deletedAt=NULL`, skip unlink |

### Schema delta (database-engineer owns; index rollout is LOAD-BEARING)

```diff
model Media {
  ...
  deletedAt          DateTime?     // already exists — no column add
+ @@index([deletedAt])
}

model Post {
  ...
  @@index([state])
  @@index([publishDate])
+ @@index([state, publishDate, deletedAt])
}
```

**No new columns.** Two new indexes. The composite Post index accelerates the JSON-LIKE eligibility scan by pre-filtering `state='PUBLISHED' AND deletedAt IS NULL` before the residual `LIKE '%mediaId%'` scan.

### Eligibility predicate (locked — 6 conditions, ALL must hold)

A Media row `m` is eligible for soft-delete when:

1. `m.deletedAt IS NULL`
2. NO `User` row references `m` via `pictureId` (FK never-delete)
3. NO `SocialMediaAgency` row references `m` via `logoId` (FK never-delete)
4. **ZERO** non-PUBLISHED Post references (`state ∈ {DRAFT, QUEUE, ERROR}` AND `Post.image LIKE '%"<m.id>"%'`)
5. AT LEAST ONE PUBLISHED Post reference (`state='PUBLISHED'` AND `deletedAt IS NULL` AND `image LIKE '%"<m.id>"%'`)
6. `MAX(publishDate)` over the predicate-5 set `<= cutoff` (cutoff = pass-start time minus 7 days)

### Canonical eligibility SQL (one round-trip per candidate, conditional aggregates)

```sql
SELECT
  MAX(p."publishDate") FILTER (WHERE p."state" = 'PUBLISHED') AS "lastPublishedAt",
  COUNT(*) FILTER (WHERE p."state" = 'PUBLISHED')::int        AS "publishedCount",
  COUNT(*) FILTER (WHERE p."state" <> 'PUBLISHED')::int       AS "nonPublishedCount"
FROM "Post" p
WHERE p."deletedAt" IS NULL
  AND p."image" LIKE ${'%"' + mediaId + '"%'}
```

Returns three fields. Interpretation:
- **Soft-delete eligible** iff `nonPublishedCount = 0` AND `publishedCount >= 1` AND `lastPublishedAt <= cutoff`
- **Hard-delete re-check eligible** iff `nonPublishedCount = 0` AND `publishedCount >= 1` (cutoff already proven by `Media.deletedAt <= cutoff - 7d`)

### Two-phase state machine

```
                      Phase 1 (soft-delete sweep): predicates 1–6 all hold
        ┌──────────────────────────────────────────────────────────────┐
        v                                                                |
   [ S0 Live ]  ─── UPDATE deletedAt = passCutoff WHERE id=? ─────> [ S1 Soft-deleted ]
        ^                                                                |
        |                                                                |
        |     Phase 2 (hard-delete sweep) intra-txn re-check FAILS       |
        └───────── UPDATE deletedAt = NULL (RESURRECT + WARN log) ───────┘
                                                                         |
                       Phase 2 re-check PASSES AND deletedAt <= now-7d   v
                                                              [ S2 Hard-deleted (row gone) ]
                                                              + post-commit: unlink file (ENOENT-safe)
```

### Concurrency / isolation (database-engineer LOCKED)

**Isolation**: `REPEATABLE READ` (downgraded from SERIALIZABLE — `FOR UPDATE` row-lock + intra-txn re-check is sufficient and avoids `SQLSTATE 40001` retries).

**Per-candidate hard-delete sequence**:
1. `BEGIN ISOLATION LEVEL REPEATABLE READ`
2. `SELECT id, path, deletedAt FROM Media WHERE id=$1 AND deletedAt IS NOT NULL AND deletedAt <= $2 FOR UPDATE` (empty → `ROLLBACK`, log `skipped-race`)
3. Re-check FK relations (`tx.user.count({pictureId: id})` + `tx.socialMediaAgency.count({logoId: id})`) — non-zero → `resurrectMedia(tx, id)` + `COMMIT` + log `resurrected-fk-relinked`
4. `getReferenceStatus(tx, id)` via the canonical SQL — interpret:
   - `nonPublishedCount > 0` → resurrect + log `resurrected-nonpub-ref`
   - `publishedCount = 0` → resurrect + log `resurrected-no-pub-ref`
   - Else → `DELETE FROM Media WHERE id=$1` + `COMMIT`
5. **Post-commit** (outside txn): `IUploadProvider.removeFile(absolutePath)`. ENOENT = success. Other errors → log `unlink.failed`, never rethrow.

### Path-confinement contract (two-layer defense, security-engineer ratified)

**Shared helper** — `libraries/nestjs-libraries/src/upload/path.confinement.ts`:

```typescript
export async function confineAndVerify(
  mediaPath: string,
  uploadRoot: string,
): Promise<
  | { ok: true; absolutePath: string }
  | { ok: false; reason: 'traversal' | 'symlink' | 'non_regular_file' | 'realpath_failed' | 'control_char' | 'unsupported_scheme' }
>
```

**Three in-the-wild `Media.path` shapes** (preparer surfaced + architect locked v2):

| Shape | Example | Resolver classification | Disposition |
|---|---|---|---|
| **Modern URL** | `${FRONTEND_URL}/uploads/2026/05/21/<hex>.png` | `kind: 'local'` | Confine + unlink |
| **Legacy relative** | `/2026/05/21/<hex>.png` (no scheme, leading slash, `posts.service.ts:188-200` verified) | `kind: 'local'` | Confine + unlink |
| **Remote http(s)** | `${CLOUDFLARE_BUCKET_URL}/<key>` (non-`FRONTEND_URL` prefix) | `kind: 'remote'` | Row-delete only; no syscall |

**Classification order is load-bearing**: modern-URL prefix → legacy-relative shape → remote http(s) → reject.

**Algorithm** (executed at both layers):
1. Reject control chars (NUL, CR, LF, anything `<0x20` or `0x7F`)
2. Classify by shape (in order):
   - `startsWith(${FRONTEND_URL}/uploads/)` → local; extract suffix
   - `/^\/\d{4}\/\d{2}\/\d{2}\//` (legacy relative form, no scheme) → local; use path as-is as suffix
   - `/^https?:\/\//` (not local prefix) → return `kind:'remote'` (row-delete only, no syscall)
   - Else → reject `unsupported_scheme` / `unknown_shape`
3. `path.resolve(uploadRoot, decodedSuffix)` → flattens `..` AND collapses legacy edge cases like `/year//month` or `/./month`
4. `path.relative(uploadRoot, target)` — reject if starts with `..` + sep, equals `..`, or is absolute
5. `fs.promises.realpath(target)` — catches symlinked intermediate dirs
6. Re-run step 4 on realpath result (re-confine)
7. `fs.promises.lstat(real)` — require `isFile()` (rejects symlinks at leaf, dirs, fifos, devices)
8. Only then return `{ ok: true, absolutePath: real }`

**Layer 1 — `MediaPathResolver`** (pre-flight, janitor caller): wraps `confineAndVerify` + classifies remote URLs. Provides attribution-rich audit logs (`runId`, `mediaId`, structured reason). Janitor skips per-row work cleanly without catching adapter exceptions.

**Layer 2 — `LocalStorage.removeFile`** (authoritative gate, defense-in-depth): re-runs the confinement check against the configured `uploadDirectory`. Throws typed `PathConfinementError` on rejection. **Any future caller of `removeFile` is automatically protected**, even if it bypasses the resolver.

**Invariant**: no janitor code path may call `fs.unlink` / `fs.promises.unlink` directly. All deletions flow through `IUploadProvider.removeFile(resolution.absolutePath)` AFTER `MediaPathResolver.resolveForDelete(...)` returned `{ kind: 'local' }`. Enforceable by grep audit.

### Clock contract (load-bearing per test-engineer mandate)

```typescript
@Injectable()
export class ClockService {
  now(): Date { return new Date(); }
  nowMs(): number { return Date.now(); }
}
```

**Ban**: ESLint `no-restricted-syntax` blocks `new Date()` and `Date.now()` in:
- `libraries/nestjs-libraries/src/database/prisma/media/media.janitor.*`
- `apps/cron/src/tasks/media.janitor.ts`
- `libraries/nestjs-libraries/src/upload/media.path.resolver.ts`

ClockService impl is the only exempt site. CI grep is belt + braces. Test impl is `FakeClockService` with freezable + advanceable interface.

**Cutoff topology**: JS layer passes integer `ageDays` to the repository; raw SQL builds `NOW() - INTERVAL '${ageDays} days'` inline. No JS-computed timestamp ever crosses the JS→SQL boundary. Eliminates clock-skew entirely.

## Invariants (architect-locked; test-engineer + security-engineer ratification target)

These 10 invariants are the verification contract. Each is grep-auditable or testable. Violation = MERGE BLOCK.

1. **Path invariant**: every unlink-bound path string is produced by `MediaPathResolver.resolveForDelete(...) => { kind: 'local', absolutePath }` AND re-confined by `LocalStorage.removeFile`. No other code path may construct an unlink argument. (Grep-audited at PR review.)
2. **Clock invariant**: no `new Date()` or `Date.now()` exists in janitor files except inside `ClockService`. Enforced by ESLint + CI grep.
3. **Cutoff invariant**: no JS-computed timestamp ever crosses the JS→SQL boundary as an eligibility cutoff. Integer day-counts only; SQL builds `NOW() - ($1::int * INTERVAL '1 day')`.
4. **Transaction invariant**: hard-delete row mutations happen INSIDE a REPEATABLE READ txn with `FOR UPDATE` row-locking. Unlinks happen AFTER commit. dryRun ROLLBACKs at txn end.
5. **Idempotency invariant**: re-running a tick is safe under all states. Soft-delete UPDATE filtered by `deletedAt IS NULL`. Hard-delete locks rows. Unlink ENOENT = success.
6. **Storage-backend invariant**: janitor is INERT (zero side effects) when `STORAGE_PROVIDER !== 'local'`. Guarded at task entry.
7. **Symlink invariant**: any symlink at leaf is rejected (lstat). Any intermediate symlink whose realpath escapes `uploadRoot` is rejected (realpath → re-confine).
8. **Three-shape classification invariant**: resolver correctly classifies modern URL / legacy relative / remote http(s) shapes per the ordered classification table; all other shapes rejected with a typed reason.
9. **Resurrect invariant**: when phase-2 re-check fails, `UPDATE Media SET deletedAt = NULL` (resurrect) is mandatory. NEVER `DELETE` on a re-check failure.
10. **Disk-orphan invariant**: unlink failure post-commit logs at WARN but never rolls back the row delete. Dangling-row (unlink ok, DELETE rolled back) is NOT acceptable; disk-orphan is.
